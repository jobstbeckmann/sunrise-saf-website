Query and Concretise

The Query and Concretise component of the Safety Assurance Framework serves two main functions: retrieving scenarios from the SUNRISE federated layer and combining them with testing objectives and other external requirements to create individual test cases. The following outlines the workflow of this block, though further refinement will occur after the mid-term review. First, the input, which includes test objectives, Operational Design Domain (ODD) and behavioral requirements, along with external requirements, is fed into the process for creating concrete test scenarios with corresponding objectives. These external requirements are then used to query individual Scenario Databases (SCDBs) via the SUNRISE federated layer. The queried SCDBs return logical or concrete scenarios, which are then adapted for testing purposes. If the scenarios returned are at the logical level, meaning they contain parameters described within ranges, the process proceeds in two steps. First, concrete scenarios with specific parameter values are created. Second, the concrete test scenarios are combined with the test objectives for further use. If the scenarios returned are already in concrete form, the process skips the creation step and moves directly to combining the scenarios with the test objectives. Once this is done, the block sends the fully developed concrete test scenarios, along with their objectives, to the environment allocation block for execution. During or after execution, feedback is provided through the analysis block, which can then generate new sets of concrete parameter combinations. For instance, an optimization algorithm could be applied to explore specific testing objectives, such as identifying failure points within a logical scenario. This workflow ensures an efficient and dynamic process for scenario creation and refinement, facilitating effective testing.
