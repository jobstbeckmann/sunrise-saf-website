Scenario Storage

Scenario storage involves two key components: connections to individual Scenario Databases (SCDBs) and the SUNRISE federated layer. This is where the outcomes of the SUNRISE project are incorporated, starting with the integration and requirements of the federated system. The process begins with the input layer, which provides necessary requirements such as the Operational Design Domain (ODD) and behavioral criteria. These requirements are then sent to the federated layer, which acts as an intermediary. The federated layer queries the individual SCDBs, retrieving scenarios that align with the specified requirements. By facilitating this interaction between the input layer, federated layer, and SCDBs, the scenario storage system ensures that the correct scenarios are selected and available for testing, in alignment with the defined objectives and criteria.
